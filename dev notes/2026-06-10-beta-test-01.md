Area: Radar Search
Type: UX
Severity: Medium

Repro:
Open Radar search controls.

Expected:
Search purpose is immediately obvious.

Actual:
Dropdown contains Name / Set # / Oracle / Type but gives no indication of what it controls until opened. Creates visual clutter and decision-making before search.

Thought:
Consider removing search mode entirely and using a unified smart search. User should be able to type:
- Sol Ring
- MH3 123
- Counterspell
- Amonkhet Invocation

and let the search logic determine intent.

Area: Radar → Buy Workflow
Type: Workflow
Severity: High

Repro:
Attempt to buy a Radar card.

Expected:
Ability to enter desired quantity before purchase.

Actual:
Buy action appears to be single-click and immediate.

Thought:
Even in a simulation, buying multiple copies is a core workflow. User should be able to specify quantity during purchase instead of repeatedly buying one at a time.

Area: Radar
Type: Feature Gap
Severity: High

Repro:
Track a card before entering.

Expected:
Support watchlist scenarios where user is waiting for a price target, time window, or confirmation signal before buying.

Actual:
Radar does not appear to support planned entry criteria beyond simply watching the card.

Thought:
Many specs are not immediate buys. User may want:
- Buy below $X
- Buy within next Y weeks if price stabilizes
- Buy if price does NOT fall further
- Buy after specific catalyst/event

Radar should support planned entry thesis, not only card tracking.

Area: Radar → Positions
Type: Workflow
Severity: Medium

Repro:
Buy a card from Radar.

Expected:
Radar item remains visible or user can choose whether to remove it.

Actual:
Card immediately disappears from Radar and moves to Positions.

Thought:
Users may continue monitoring a card after initial entry, especially when scaling into positions. Automatic removal feels abrupt and may break workflow.

Area: Global Notifications
Type: UX
Severity: Medium

Repro:
Save targets, hold times, or other data.

Expected:
Save confirmation appears without affecting layout.

Actual:
Page displays save message near header and causes visible page movement/jump.

Thought:
Replace layout-shifting save messages with toast notifications. Standard pattern:
- Small popup in corner
- Auto-dismiss after several seconds
- Manual dismiss option
- No page movement
