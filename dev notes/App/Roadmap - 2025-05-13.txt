🔥 Architectural Decisions That Changed The Project
1. Terminal-style interaction direction

Dropdown-heavy UI abandoned.

Replaced with:

header-click sorting
fast workflows
lower friction interaction

ManaSpec now behaves more like:

a lightweight trading terminal

than:

a form-driven web app

2. State-driven render architecture

Core pattern established:

state → render → interaction → state update → re-render

This is now the foundation for:

watchlists
alerts
dashboard signals
opportunity systems
3. Pricing abstraction direction established

Critical future decision:

App systems should ask:

“what is the current price?”

NOT:

“where specifically does this price come from?”

This enables future:

TCGPlayer integration
Cardmarket integration
cached pricing
historical snapshots
blended market logic
arbitrage systems

Without rewriting business logic.

4. ManaSpec product identity clarified

ManaSpec is NOT:

inventory software
collection nostalgia software
rapid flip/day-trading software

ManaSpec IS becoming:

a long-horizon MTG speculation intelligence terminal

Focused on:

conviction specs
print-level precision
decision tracking
market signal interpretation
long-term learning loops
📊 Current Core Workflow
Find card
→ inspect printings
→ evaluate versions
→ execute spec
→ monitor over time
→ review thesis/results

This is now the app’s central loop.
