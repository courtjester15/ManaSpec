# Dev Notes

This folder is project memory, not the active source of truth.

Use this structure:

- `inbox/`: current daily notes, GPT brainstorming, beta-test observations, and rough session notes.
- `archive/`: older historical notes kept for reference after their useful ideas have been promoted or superseded.

Rules:

- Put new raw notes in `inbox/` with a date-first filename.
- Promote durable product behavior to `docs/README.md`.
- Promote priorities and checklists to `docs/ROADMAP.md`.
- Promote durable decisions and rationale to `docs/DECISIONS.md`.
- Do not create module subfolders here unless the notes folder itself becomes hard to scan again.
- If notes conflict with active docs, active docs win.
