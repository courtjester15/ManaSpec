AI Dev Workflow (ManaSpec Notes System)

Active docs now live in:

docs/

Start with:

- docs/README.md
- docs/WORKFLOW.md
- docs/ROADMAP.md
- docs/DECISIONS.md
- docs/product/spec.md

Core idea:

Use AI to build software efficiently by separating:

- thinking: product direction and design choices
- execution: code changes based on current specs
- state: what is true now

Folder model:

docs/
  product/
    spec.md
  modules/
    portfolio/
      spec.md
    search/
      spec.md
  DECISIONS.md

What each file means:

spec.md = source of truth

- current implementation
- how the feature works now
- rules and structure
- what AI should follow when coding

DECISIONS.md = why things are the way they are

- key design decisions
- tradeoffs
- choices that should not be relitigated casually

dev notes/inbox = raw current notes
dev notes/archive = memory

- daily brainstorming
- beta observations
- old drafts and outdated ideas

Key rules:

- Do not ask AI to read the whole repo or all notes by default.
- Point AI to one module and its spec whenever possible.
- Active docs in docs/ win over historical notes.
- Notes in dev notes/inbox/ and dev notes/archive/ are reference only.
- Update spec.md when behavior changes.
- Update docs/DECISIONS.md when a meaningful choice is made.

Mental model:

- spec.md = reality
- DECISIONS.md = reasoning
- dev notes = memory and raw intake
- AI = executes current reality, not history
